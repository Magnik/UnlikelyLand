# UnlikelyLand

A persistent, text-driven, weird-comedy multiplayer RPG for the browser (and, later, an installable PWA / Android app). You wash up on a strange island, spend regenerating stamina on absurd expeditions, make choices that quietly shape who your character becomes, fight the occasional militant recliner, sometimes die, and slowly work toward escaping.

The defining rule: **the server is authoritative and the AI only ever proposes.** Encounters can be generated by a local LLM, but every reward, stat change, combat result, and death is decided by server-side game rules. The game is fully playable with the AI switched off.

> New here as a developer? Start with [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

## Stack

| Layer | Choice |
|---|---|
| Frontend | Next.js 14 (App Router) + React, mobile-first, PWA manifest |
| Backend | NestJS + TypeScript |
| Database | PostgreSQL + Prisma ORM |
| Validation | Zod (shared `@unlikelyland/contracts` package) |
| Auth | Username/password, JWT (bcrypt hashing) |
| AI | Ollama-compatible provider behind a swappable gateway |
| Realtime/jobs | Redis (reserved for chat + BullMQ in later milestones) |
| Deploy | Docker Compose + Caddy (automatic HTTPS) |

Monorepo layout:

```
packages/contracts   # shared zod schemas + types (enums, encounter.v1, API DTOs)
apps/api             # NestJS game server (engine, AI gateway, REST API)
apps/web             # Next.js client (the whole UI)
docs/                # architecture, game design, deployment, roadmap, Android notes
deploy/, scripts/    # Caddy config, Postgres backup/restore
```

## Prerequisites

- Node.js 20+
- Docker + Docker Compose (for the containerised path)
- (Optional) [Ollama](https://ollama.com) for AI-generated encounters — the game works without it.

## Quick start — Docker (recommended)

```bash
cp .env.example .env
# Edit .env: set strong POSTGRES_PASSWORD, JWT_SECRET, ADMIN_PASSWORD.
docker compose up --build
```

- Web app: <http://localhost:3000>
- API: <http://localhost:4000> (health at `/health`)

The API container runs migrations and seeds the world (region sets, item catalog, achievements, AI settings, and the admin account) automatically on first start. Register a player at `/register`, or log in as the seeded admin (`ADMIN_USERNAME` / `ADMIN_PASSWORD`) to reach `/admin`.

## Quick start — local (no Docker)

You need a PostgreSQL instance. Point `DATABASE_URL` at it (Prisma reads `.env` from `apps/api/`, so copy the root `.env` there too):

```bash
cp .env.example .env
cp .env apps/api/.env        # Prisma CLI reads env from the workspace dir
npm install
npm run build:contracts      # build the shared package first

# Database
npm run prisma:migrate -w @unlikelyland/api   # create/apply migrations
npm run prisma:seed -w @unlikelyland/api      # seed the world + admin

# Run the two apps (separate terminals)
npm run dev:api              # NestJS on :4000
npm run dev:web              # Next.js on :3000
```

## AI (Ollama) — optional

The AI gateway talks to any Ollama-compatible `/api/chat` endpoint.

1. Install Ollama and pull a model: `ollama pull llama3.1:8b`
2. In `.env` set `OLLAMA_BASE_URL` and `OLLAMA_MODEL`.
   - Running the API in Docker but Ollama on your host? Use `http://host.docker.internal:11434` (already wired via `extra_hosts`).
3. AI can be toggled at runtime from the Admin dashboard (Enable/Disable AI, Force fallback). Every generation attempt is validated against `encounter.v1`, moderated, and logged; on any failure or timeout the server falls back to seeded content. **The game never breaks when the AI is offline.**
